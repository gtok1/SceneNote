SceneNote 앱의 프로젝트 오너이자 주도 개발자. 소규모 팀(1-3명) 기준 MVP를 목표로 한다.

기술 스택(Expo + Supabase + TanStack Query + Zustand/Jotai)을 직접 선정하였으며, 아키텍처 문서(01~09)를 체계적으로 작성할 만큼 문서화 주도적.

핵심 가치: "기록이 먼저다. 검색은 도구일 뿐이다." — 타임라인 핀 경험을 최우선으로 설계 방향을 잡고 있음.

**Why:** 소규모 팀이므로 복잡한 마이크로서비스보다 Supabase + Edge Functions 기반의 현실적인 구조를 선호.
**How to apply:** 아키텍처 제안 시 항상 1-3인 팀 기준 운영 가능성을 기준으로 복잡도를 평가할 것.

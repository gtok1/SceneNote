# SceneNote — 유저 스토리 문서

**버전:** 1.0.0
**작성일:** 2026-05-02
**작성자:** MVP Product Planner
**상태:** 검토 중

---

## 개요

이 문서는 SceneNote MVP의 유저 스토리를 정의한다. 각 스토리는 실제 사용자의 행동 흐름을 기반으로 작성되었으며, 디자이너·프론트엔드·백엔드가 구현 기준으로 사용할 수 있도록 Acceptance Criteria를 구체적으로 명시했다.

### 스토리 번호 체계

| 번호 범위 | 분류 |
|-----------|------|
| US-001 ~ US-009 | 인증 및 사용자 관리 |
| US-010 ~ US-019 | 콘텐츠 검색 |
| US-020 ~ US-029 | My Library 관리 |
| US-030 ~ US-039 | 에피소드 진행률 |
| US-040 ~ US-059 | 타임라인 핀 |
| US-060 ~ US-069 | 태그 / 필터링 |
| US-070 ~ US-079 | My Page |

## US-001: 이메일로 회원가입

**역할:** 신규 사용자
**상황:** SceneNote를 처음 설치하고 시작하려는 사용자
**목표:** 이메일과 비밀번호로 계정을 만들고 앱을 사용하기 시작한다
**행동:** 온보딩 화면에서 "이메일로 시작하기"를 탭하고, 이메일과 비밀번호를 입력한 후 가입 버튼을 누른다
**기대 결과:** 계정이 생성되고 홈 화면으로 이동한다

**Acceptance Criteria:**
1. 이메일 형식이 올바르지 않으면 인라인 오류 메시지를 표시한다.
2. 비밀번호가 8자 미만이면 오류 메시지를 표시한다.
3. 이미 가입된 이메일이면 "이미 사용 중인 이메일입니다" 메시지를 표시한다.
4. 가입 성공 시 이메일 인증 안내 없이 즉시 로그인 상태가 된다. [확실하지 않음 — 이메일 인증 필요 여부 추후 검토]
5. 가입 완료 후 홈 화면으로 이동한다.
6. 비밀번호 입력 필드에는 보기/숨기기 토글이 제공된다.

**우선순위:** P0
**관련 화면:** 온보딩, 회원가입

---

## US-002: 소셜 계정으로 로그인

**역할:** 기존 사용자 또는 신규 사용자
**상황:** 이메일 가입 없이 빠르게 시작하려는 사용자
**목표:** Google 또는 Apple 계정으로 로그인한다
**행동:** 온보딩 화면에서 "Google로 계속하기" 또는 "Apple로 계속하기"를 탭한다
**기대 결과:** 소셜 인증 흐름이 완료되고 홈 화면으로 이동한다

**Acceptance Criteria:**
1. Google OAuth 인증이 정상 동작한다.
2. Apple Sign-In이 정상 동작한다 (iOS 기기에서 필수 노출).
3. 소셜 로그인 최초 사용 시 자동으로 계정이 생성된다.
4. 소셜 로그인 재사용 시 기존 계정으로 로그인된다.
5. 소셜 인증 취소 시 온보딩 화면으로 돌아간다.
6. 소셜 인증 실패 시 "로그인에 실패했습니다. 다시 시도해 주세요" 메시지를 표시한다.

**우선순위:** P1
**관련 화면:** 온보딩, 로그인

## US-003: 로그인 (재방문 사용자)

**역할:** 기존 사용자
**상황:** 앱을 재실행하거나 로그아웃 후 다시 접속하는 사용자
**목표:** 기존 계정으로 로그인하여 내 기록에 접근한다
**행동:** 로그인 화면에서 이메일과 비밀번호를 입력하고 로그인 버튼을 탭한다
**기대 결과:** 홈 화면으로 이동하고 이전에 저장한 라이브러리와 핀이 보인다

**Acceptance Criteria:**
1. 올바른 이메일/비밀번호 입력 시 즉시 로그인된다.
2. 잘못된 자격증명 시 "이메일 또는 비밀번호가 올바르지 않습니다" 메시지를 표시한다.
3. 세션이 유효한 경우 앱 재실행 시 자동으로 로그인 상태를 유지한다.
4. "로그인 상태 유지"가 기본값으로 활성화되어 있다. [확실하지 않음 — 추후 검토]

**우선순위:** P0
**관련 화면:** 로그인

## US-010: 작품 검색

**역할:** 로그인된 사용자
**상황:** 특정 작품을 My Library에 추가하거나 정보를 확인하려는 사용자
**목표:** 작품명을 검색하여 원하는 작품을 찾는다
**행동:** 검색 탭을 탭하고 검색창에 작품명을 입력한다
**기대 결과:** 입력한 키워드와 관련된 작품 목록이 표시된다

**Acceptance Criteria:**
1. 검색창 포커스 시 키보드가 자동으로 올라온다.
2. 키워드 입력 후 검색 버튼(또는 키보드 Return) 탭 시 검색이 실행된다.
3. 검색 결과는 최대 20건이 첫 페이지에 표시된다.
4. 각 검색 결과 카드에는 포스터 이미지, 작품명, 연도, 콘텐츠 유형, 출처 API 레이블이 표시된다.
5. 검색 중에는 로딩 인디케이터를 표시한다.
6. 네트워크 오류 시 "검색에 실패했습니다. 다시 시도해 주세요" 메시지와 재시도 버튼을 표시한다.
7. 외부 API 키는 클라이언트에 노출되지 않는다 (Edge Functions 경유).

**우선순위:** P0
**관련 화면:** 콘텐츠 검색, 검색 결과

---

## US-011: 검색 결과 없음

**역할:** 로그인된 사용자
**상황:** 철자 오류 또는 등록되지 않은 작품을 검색한 사용자
**목표:** 검색 결과가 없을 때 상황을 파악하고 대안을 찾는다
**행동:** 존재하지 않는 작품명이나 오타가 있는 키워드를 입력하고 검색한다
**기대 결과:** 검색 결과 없음 상태가 명확하게 표시되고 다음 행동을 안내받는다

**Acceptance Criteria:**
1. 모든 외부 API에서 결과가 없을 경우 빈 상태 일러스트와 안내 메시지를 표시한다.
2. 안내 메시지: "검색 결과가 없습니다. 다른 키워드로 검색해 보세요."
3. 검색창에 기존 키워드가 유지되어 수정이 용이하다.
4. 검색 결과 없음 상태는 네트워크 오류 상태와 UI가 구분된다.

**우선순위:** P1
**관련 화면:** 검색 결과

---

## US-012: 검색 결과 페이지네이션

**역할:** 로그인된 사용자
**상황:** 인기 작품(예: "드래곤")처럼 결과가 많은 키워드를 검색한 사용자
**목표:** 처음 20건 이후의 결과도 탐색한다
**행동:** 검색 결과 목록을 끝까지 스크롤한다
**기대 결과:** 추가 결과가 자동으로 로드된다

**Acceptance Criteria:**
1. 결과 목록 하단에 도달하면 자동으로 다음 페이지를 요청한다 (무한 스크롤).
2. 다음 페이지 로딩 중 하단에 로딩 스피너를 표시한다.
3. 더 이상 결과가 없으면 "모든 결과를 불러왔습니다" 안내를 표시한다.
4. 네트워크 오류 시 재시도 버튼을 표시한다.

**우선순위:** P1
**관련 화면:** 검색 결과

---

## US-013: 콘텐츠 상세 정보 조회

**역할:** 로그인된 사용자
**상황:** 검색 결과에서 특정 작품을 탭한 사용자
**목표:** 작품의 상세 정보를 확인하고 라이브러리 추가 여부를 결정한다
**행동:** 검색 결과 카드를 탭한다
**기대 결과:** 해당 작품의 상세 화면으로 이동한다

**Acceptance Criteria:**
1. 작품 포스터, 제목, 원제, 장르, 연도, 줄거리 요약이 표시된다.
2. 시리즈인 경우 시즌 수와 총 에피소드 수가 표시된다.
3. 이미 라이브러리에 추가된 작품인 경우 현재 감상 상태가 표시된다.
4. "라이브러리에 추가" 버튼이 항상 상단에 고정되어 있다.
5. 이미 추가된 경우 버튼 텍스트가 "라이브러리에서 보기"로 변경된다.

**우선순위:** P1
**관련 화면:** 콘텐츠 상세

## US-020: 작품을 My Library에 추가

**역할:** 로그인된 사용자
**상황:** 콘텐츠 상세 화면에서 마음에 드는 작품을 발견한 사용자
**목표:** 작품을 My Library에 추가하고 감상 상태를 설정한다
**행동:** "라이브러리에 추가" 버튼을 탭하고 바텀 시트에서 감상 상태를 선택한다
**기대 결과:** 작품이 My Library에 저장되고 선택한 상태로 기록된다

**Acceptance Criteria:**
1. "라이브러리에 추가" 버튼 탭 시 감상 상태 선택 바텀 시트가 표시된다.
2. 선택 옵션: "보고 싶음", "보는 중", "완료", "드랍"
3. 상태 선택 후 즉시 저장되고 바텀 시트가 닫힌다.
4. 저장 완료 후 콘텐츠 상세 화면의 버튼이 "라이브러리에서 보기"로 변경된다.
5. 동일 작품을 두 번 추가하려 하면 "이미 라이브러리에 있는 작품입니다" 토스트를 표시한다.
6. 저장 실패 시 "저장에 실패했습니다. 다시 시도해 주세요" 토스트를 표시한다.
7. 작품 메타데이터(제목, 포스터 URL, 외부 ID 등)가 contents 테이블에 저장된다.

**우선순위:** P0
**관련 화면:** 콘텐츠 상세, My Library에 추가 모달

## US-021: 감상 상태 변경

**역할:** 로그인된 사용자
**상황:** 시청을 시작하거나 완료한 작품의 상태를 업데이트하려는 사용자
**목표:** 라이브러리의 작품 감상 상태를 변경한다
**행동:** 라이브러리 목록 또는 작품 상세에서 현재 상태 배지를 탭하고 새 상태를 선택한다
**기대 결과:** 감상 상태가 즉시 업데이트된다

**Acceptance Criteria:**
1. 현재 상태 배지 탭 시 상태 변경 바텀 시트가 표시된다.
2. 현재 상태는 선택된 상태로 표시된다.
3. 새 상태 선택 시 즉시 저장되고 바텀 시트가 닫힌다.
4. `completed` → `watching` 전환 시 "다시 시청을 시작하시겠습니까?" 확인 팝업을 표시한다.
5. 상태 변경은 타임라인 핀 데이터에 영향을 주지 않는다.
6. 변경 실패 시 이전 상태로 롤백하고 오류 토스트를 표시한다.

**우선순위:** P1
**관련 화면:** 홈/라이브러리, 콘텐츠 상세

## US-022: My Library 전체 조회

**역할:** 로그인된 사용자
**상황:** 내가 기록한 모든 작품을 한눈에 확인하려는 사용자
**목표:** 라이브러리에 저장된 모든 작품을 탐색한다
**행동:** 홈 탭을 탭하여 My Library 화면으로 진입한다
**기대 결과:** 저장된 모든 작품이 상태별로 확인 가능하다

**Acceptance Criteria:**
1. 홈 탭 진입 시 My Library 목록이 기본으로 표시된다.
2. 감상 상태별 탭 또는 필터 (전체 / 보는 중 / 완료 / 보고 싶음 / 드랍)가 제공된다.
3. 각 작품 카드에는 포스터 썸네일, 제목, 감상 상태 배지, 최근 핀 수가 표시된다.
4. 라이브러리가 비어있는 경우 빈 상태 UI와 "작품을 검색해서 추가해보세요" 안내를 표시한다.
5. 작품 카드 탭 시 현재 시청 중 상세 화면으로 이동한다.

**우선순위:** P0
**관련 화면:** 홈/라이브러리

---

## US-023: 라이브러리에서 작품 제거

**역할:** 로그인된 사용자
**상황:** 더 이상 관리하지 않을 작품을 라이브러리에서 삭제하려는 사용자
**목표:** 작품을 My Library에서 삭제한다
**행동:** 작품 카드를 길게 누르거나 상세 화면에서 "라이브러리에서 제거" 옵션을 선택한다
**기대 결과:** 작품이 라이브러리에서 삭제된다

**Acceptance Criteria:**
1. 삭제 전 "이 작품을 라이브러리에서 제거하시겠습니까? 관련된 핀 기록도 함께 삭제됩니다." 확인 팝업을 표시한다.
2. 확인 후 `user_library_items`, `user_episode_progress`, `timeline_pins` 관련 레코드가 삭제된다.
3. 삭제 완료 후 라이브러리 목록에서 해당 작품이 즉시 사라진다.
4. 취소 선택 시 삭제가 이루어지지 않는다.
5. 삭제 실패 시 오류 토스트를 표시한다.
6. contents 테이블의 콘텐츠 메타데이터는 삭제하지 않는다. [Tech Lead 검토 필요] 다른 사용자도 동일 콘텐츠를 사용할 수 있으므로 공유 메타데이터 삭제 정책 확인 필요.

**우선순위:** P1
**관련 화면:** 홈/라이브러리, 콘텐츠 상세

## US-030: 에피소드 시청 완료 체크

**역할:** 로그인된 사용자
**상황:** 특정 에피소드를 다 보고 나서 진행률을 기록하려는 사용자
**목표:** 시청한 에피소드를 완료 상태로 체크한다
**행동:** 에피소드 선택 화면에서 해당 에피소드의 체크박스를 탭한다
**기대 결과:** 에피소드가 완료 상태로 표시되고 진행률이 업데이트된다

**Acceptance Criteria:**
1. 에피소드 체크박스 탭 시 완료 상태로 즉시 표시된다.
2. 이미 완료된 에피소드 체크박스 재탭 시 완료 취소(미시청 상태)로 돌아간다.
3. `user_episode_progress` 테이블에 해당 에피소드 레코드가 저장된다.
4. 시즌 전체 진행률(예: "12/24화 완료")이 시즌 헤더에 업데이트된다.
5. 마지막 에피소드 완료 체크 시 "전체 시청 완료! 감상 상태를 '완료'로 변경할까요?" 팝업을 표시한다.
6. 감상 상태가 `wishlist`인 상태에서 에피소드를 체크하면 자동으로 `watching`으로 변경한다. [확실하지 않음 — 추후 검토]

## US-040: 타임라인 핀 생성 (기본)

**역할:** 로그인된 사용자
**상황:** 인상적인 장면을 시청하고 해당 순간을 기록하려는 사용자
**목표:** 특정 에피소드의 특정 시간에 핀을 생성한다
**행동:** 에피소드 선택 화면에서 "핀 추가" 버튼을 탭하고 시간과 메모를 입력한 후 저장한다
**기대 결과:** 핀이 생성되고 해당 에피소드의 핀 목록에 표시된다

**Acceptance Criteria:**
1. 핀 생성 화면에서 타임스탬프 입력 필드와 메모 입력 필드가 제공된다.
2. 타임스탬프는 MM:SS 또는 HH:MM:SS 형식으로 입력한다.
3. 입력된 타임스탬프는 `timestamp_seconds`(정수)로 변환되어 저장된다.
4. 메모는 필수 항목이 아니다. 타임스탬프만으로도 핀 생성이 가능하다. [확실하지 않음 — 메모 필수 여부 추후 검토]
5. 저장 버튼 탭 시 `timeline_pins` 테이블에 레코드가 생성된다.
6. 저장 완료 후 핀 목록 화면으로 이동하거나 에피소드 화면으로 돌아간다.
7. 동일 에피소드에 복수의 핀 생성이 허용된다.
8. 동일 시간값에 복수의 핀 생성이 허용된다.

**우선순위:** P0
**관련 화면:** 타임라인 핀 생성, 에피소드 선택

## US-042: 타임라인 핀에 감정 추가

**목표:** 핀에 감정 이모티콘/레이블을 선택하여 저장한다
**행동:** 핀 생성/편집 화면에서 감정 선택 옵션 중 하나를 탭한다
**기대 결과:** 선택한 감정이 핀에 저장되고 핀 목록에서 표시된다

**Acceptance Criteria:**
1. 감정 선택 UI는 수평 스크롤 가능한 감정 목록으로 제공된다.
2. 선택 가능한 감정 목록은 사전 정의된 항목이다 (09_timeline_pin_ux.md 참조).
3. 감정은 하나만 선택 가능하다. 이미 선택된 감정을 다시 탭하면 선택 해제된다.
4. 감정을 선택하지 않아도 핀 저장이 가능하다 (선택 사항).
5. 핀 목록에서 감정 이모티콘이 핀 카드에 표시된다.
6. 선택된 감정은 `timeline_pins.emotion` 컬럼에 문자열로 저장된다.

## US-043: 타임라인 핀에 스포일러 플래그 설정

**상황:** 나중에 다시 봤을 때 내용을 미리 보고 싶지 않은 핀을 만드는 사용자
**목표:** 핀에 스포일러 플래그를 설정하여 목록에서 블러 처리되게 한다

**Acceptance Criteria:**

4. 핀 목록에서 스포일러 핀의 메모 텍스트는 블러 처리된다.
5. 스포일러 핀 탭 또는 "보기" 버튼 탭 시 내용이 공개된다.
6. 스포일러 핀은 목록에서 스포일러 아이콘/레이블로 구분된다.

5. 스포일러 핀의 메모는 블러 처리된다.
6. 핀이 없는 경우 빈 상태 UI를 표시한다.

## US-046: 타임라인 핀 수정

**역할:** 로그인된 사용자
**상황:** 이전에 저장한 핀의 내용을 수정하려는 사용자
**목표:** 기존 핀의 타임스탬프, 메모, 태그, 감정, 스포일러 여부를 수정한다

**기대 결과:** 수정된 내용이 즉시 저장되고 목록에 반영된다

**상황:** 라이브러리에 저장된 특정 작품을 빠르게 찾으려는 사용자
**목표:** 내 라이브러리 안에서 작품 제목으로 검색한다
**행동:** 라이브러리 화면 상단의 검색 아이콘을 탭하고 작품명을 입력한다
**기대 결과:** 내 라이브러리 내에서 제목이 일치하는 작품만 필터링되어 표시된다

**Acceptance Criteria:**
1. 검색 입력 시 실시간으로 라이브러리가 필터링된다.
2. 검색은 로컬 데이터 기준으로 동작한다 (외부 API 호출 없음).
3. 검색 결과가 없으면 "검색 결과가 없습니다" 안내를 표시한다.
4. 검색창 X 버튼 또는 취소 탭 시 검색이 해제되고 전체 목록이 표시된다.

**우선순위:** P2
**관련 화면:** 홈/라이브러리

## US-071: My Page 조회

**역할:** 로그인된 사용자
**상황:** 내 감상 기록 통계와 프로필을 확인하려는 사용자
**목표:** My Page에서 기본 통계와 계정 정보를 확인한다
**행동:** 하단 탭바의 프로필 아이콘을 탭한다
**기대 결과:** 나의 감상 기록 요약과 계정 설정이 표시된다

**Acceptance Criteria:**

2. 로그아웃 버튼이 제공된다.
3. 회원 탈퇴 옵션이 제공된다 (경고 메시지 포함).
4. 이메일 또는 소셜 계정 정보가 표시된다.

**우선순위:** P1
**관련 화면:** My Page

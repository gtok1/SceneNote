2026-05-02 기준 확정된 엣지 케이스 처리 결정사항.

## 확정 결정

| 케이스 | 결정 | 근거 |
|--------|------|------|
| 동일 에피소드 복수 핀 | 허용 (개수 제한 없음) | 핵심 사용 패턴 |
| 동일 시간대 복수 핀 | 허용 (경고 없음), 생성순 나열 | 같은 순간 다른 시각 기록 허용 |
| 타임스탬프 에피소드 초과 | 저장 불가, 인라인 오류 표시 | 데이터 신뢰도 유지 |
| 에피소드 런타임 정보 없음 | 초과 검증 생략, 저장 허용 | 외부 API 데이터 불완전성 대응 |
| 검색 결과 0건 | 빈 상태 UI + 키워드 수정 안내 | 사용자 다음 액션 유도 |
| 검색 결과 다수 | 20건/페이지 무한 스크롤 | 성능 + UX 균형 |
| 동일 작품 복수 API 결과 | 출처 레이블 표시, 자동 중복 제거 없음 | MVP 단순화, Phase 2에서 canonical 처리 |
| 스포일러 핀 블러 해제 | 세션 내 유지 (화면 나가면 재블러) | [확실하지 않음 — 영구 해제 옵션 추후 검토] |
| 핀 목록 정렬 | timestamp_seconds ASC, 동점 시 created_at ASC | 시간순 탐색이 핵심 UX |

## 미결 결정 (확실하지 않음)

| 케이스 | 현재 상태 | 추후 검토 내용 |
|--------|-----------|---------------|
| 타임스탬프 없는 핀 허용 여부 | 미결 | null 허용 시 목록 정렬 정책 결정 필요 |
| 메모 필수 여부 | 미결 | 타임스탬프만으로 핀 저장 허용 여부 |
| 이메일 인증 필요 여부 | 미결 | 가입 즉시 로그인 vs 이메일 인증 후 로그인 |
| 마지막 에피소드 자동 완료 전환 | 미결 | 자동 전환 vs 팝업 확인 방식 결정 필요 |
| 스포일러 영구 해제 옵션 | 미결 | 사용자 설정에서 영구 해제 옵션 추가 여부 |

**Why:** 결정된 케이스와 미결 케이스를 명확히 분리하여 개발팀 혼란 방지.
**How to apply:** 미결 항목이 결정되면 이 문서를 업데이트하고 관련 화면 정의서도 함께 수정.

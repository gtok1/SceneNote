```
    ┌──────────────────────────────────────────────────┐
    │              하단 탭바 (Tab Bar)                  │

    ↓ (작품 카드 탭)

    ↓ (키워드 입력 + 검색)

    ↓ (결과 카드 탭)

    ↓ (라이브러리에 추가 탭)

    ↓ (상태 선택 + 저장)
[SCR-006 콘텐츠 상세] → [SCR-008 현재 시청 중 상세]

[SCR-014 My Page]
    ↓ (태그 목록 탭)
[SCR-013 태그별 핀 목록]
```

## 3. 각 화면 정의

---

## SCR-001: 온보딩

- **진입 경로:** 앱 최초 실행 시, 또는 로그아웃 후 재실행 시
- **주요 UI 요소:**
  - 앱 로고 및 앱 이름 (SceneNote)
  - 핵심 가치 소개 슬라이드 (2~3장) — "내 감상 기록", "타임라인 핀"
  - "이메일로 시작하기" CTA 버튼
  - "Google로 계속하기" 버튼
  - "Apple로 계속하기" 버튼 (iOS 전용)
  - "이미 계정이 있어요" 텍스트 링크
- **사용자 액션:**
  - 슬라이드 스와이프 (선택 사항)
  - 회원가입 방법 선택
  - 로그인 화면으로 이동
- **다음 화면:**
  - 이메일 회원가입 선택 → SCR-002 (회원가입 폼)
  - 소셜 로그인 선택 → 소셜 인증 흐름 → SCR-003 (홈)
  - "이미 계정이 있어요" 탭 → SCR-002 (로그인 폼)
- **엣지 케이스:**
  - 이미 로그인된 세션이 있는 경우: 온보딩을 건너뛰고 SCR-003으로 바로 이동
  - 소셜 인증 취소 시: 온보딩으로 복귀
  - 네트워크 없는 상태에서 소셜 로그인 시도: "인터넷 연결을 확인해 주세요" 토스트 표시
- **MVP 포함 여부:** Yes — 신규 사용자 진입점이므로 필수

## SCR-002: 로그인 / 회원가입

- **진입 경로:** 온보딩에서 이메일 버튼 탭, 또는 "이미 계정이 있어요" 탭
- **주요 UI 요소:**
  - 로그인 / 회원가입 탭 전환
  - 이메일 입력 필드
  - 비밀번호 입력 필드 (보기/숨기기 토글)
  - (회원가입) 비밀번호 확인 입력 필드
  - "로그인" / "회원가입" 제출 버튼
  - "비밀번호를 잊으셨나요?" 링크 (로그인 탭)
  - 소셜 로그인 구분선 + Google/Apple 버튼
- **사용자 액션:**
  - 이메일/비밀번호 입력 및 제출
  - 비밀번호 재설정 요청
  - 소셜 로그인 선택
  - 뒤로 가기 (온보딩으로 복귀)
- **다음 화면:**
  - 인증 성공 → SCR-003 (홈/라이브러리)
  - 비밀번호 재설정 → 이메일 발송 확인 화면 (인라인 처리)
- **엣지 케이스:**
  - 이메일 형식 오류: 필드 하단 인라인 오류 메시지
  - 비밀번호 8자 미만: 인라인 오류 메시지
  - 이미 존재하는 이메일로 회원가입: "이미 사용 중인 이메일입니다" 오류
  - 잘못된 자격증명 로그인: "이메일 또는 비밀번호가 올바르지 않습니다"
  - 로그인 시도 5회 이상 실패: [Tech Lead 검토 필요] Rate limiting 정책 결정 필요
  - 네트워크 오류: "인터넷 연결을 확인해 주세요" 토스트
- **MVP 포함 여부:** Yes — 인증은 모든 기능의 전제 조건

## SCR-003: 홈 / 라이브러리

- **진입 경로:** 로그인 완료 후, 또는 하단 탭바의 홈 아이콘 탭
- **주요 UI 요소:**
  - 화면 상단: "My Library" 헤더, 검색 아이콘
  - 감상 상태 필터 탭 (전체 / 보는 중 / 완료 / 보고 싶음 / 드랍)
  - 작품 목록 (FlashList) — 각 카드: 포스터 썸네일, 제목, 상태 배지, 핀 수
  - 하단 탭바 (홈 / 검색 / My Page)
  - 빈 상태 UI (라이브러리 비어있을 때)
- **사용자 액션:**
  - 필터 탭 전환
  - 작품 카드 탭 → SCR-008로 이동
  - 검색 아이콘 탭 → 라이브러리 내 검색 활성화
  - 탭바로 다른 탭 이동
- **다음 화면:**
  - 작품 카드 탭 → SCR-008 (현재 시청 중 상세)
  - 검색 탭 → SCR-004 (콘텐츠 검색)
  - My Page 탭 → SCR-014 (My Page)
- **엣지 케이스:**
  - 라이브러리 비어있음: 빈 상태 일러스트 + "작품을 검색해서 추가해보세요" + 검색 버튼
  - 필터 결과 0건: 해당 상태에 작품 없음 안내 (다른 필터 탭은 유지)
  - 포스터 이미지 로드 실패: 기본 이미지(플레이스홀더) 표시
  - 라이브러리 항목 100개 이상: FlashList 가상화 처리 (스크롤 성능 유지)
- **MVP 포함 여부:** Yes — 앱의 메인 화면

## SCR-004: 콘텐츠 검색

- **진입 경로:** 하단 탭바의 검색 아이콘 탭
- **주요 UI 요소:**
  - 검색 입력 필드 (자동 포커스)
  - 취소 버튼
  - 콘텐츠 유형 필터 칩 (전체 / 애니 / 드라마 / 영화)
  - 검색 전 상태: 최근 검색어 목록 (P2) 또는 빈 화면 안내
  - 검색 결과 목록 (SCR-005와 동일 화면 내 상태 전환)
- **사용자 액션:**
  - 키워드 입력
  - 검색 실행 (Return 키 또는 검색 버튼)
  - 유형 필터 선택
  - 취소 → 홈으로 복귀
- **다음 화면:**
  - 검색 실행 → 동일 화면에서 결과 목록 표시 (SCR-005 상태)
  - 결과 카드 탭 → SCR-006 (콘텐츠 상세)
- **엣지 케이스:**
  - 공백만 입력 후 검색: 검색 실행하지 않음, "검색어를 입력해 주세요" 토스트
  - 특수문자 포함 입력: 그대로 API에 전달 (필터링 미수행) [확실하지 않음 — 추후 검토]
  - 검색 중 네트워크 끊김: "인터넷 연결을 확인해 주세요" + 재시도 버튼
  - 검색 중 화면 이탈 후 복귀: 이전 검색 결과 유지 (TanStack Query 캐시 활용)
- **MVP 포함 여부:** Yes — 라이브러리 추가를 위한 진입점

---

## SCR-005: 검색 결과

- **진입 경로:** SCR-004에서 검색 실행 후 (동일 화면 내 결과 상태로 처리)
- **주요 UI 요소:**
  - 검색어 표시 + 수정 가능한 검색 필드
  - 유형 필터 칩 (전체 / 애니 / 드라마 / 영화)
  - 결과 목록 (카드형) — 포스터, 제목, 원제, 연도, 유형, 출처 API 레이블
  - 로딩 스켈레톤 UI
  - 결과 없음 상태 UI
  - 페이지네이션 (하단 무한 스크롤)
- **사용자 액션:**
  - 검색어 수정 후 재검색
  - 유형 필터 변경
  - 결과 카드 탭
  - 하단 스크롤로 추가 결과 로드
- **다음 화면:**
  - 결과 카드 탭 → SCR-006 (콘텐츠 상세)
- **엣지 케이스:**
  - 결과 0건: 빈 상태 UI + "검색어를 바꿔보세요" 안내
  - 동일 작품이 여러 API에서 검색됨: 각 카드에 출처 API 레이블 표시, 중복 자동 제거 미수행 (MVP)
  - 결과 수백 건 (예: "dragon"): 페이지당 20건, 무한 스크롤로 처리
  - 모든 API 동시 실패: "검색 서비스에 문제가 생겼습니다" + 재시도 버튼
  - 일부 API만 실패: 성공한 API 결과만 표시, 하단에 "일부 결과가 표시되지 않을 수 있습니다" 배너 [확실하지 않음 — 추후 검토]
  - **[Tech Lead 검토 필요]** 외부 API 호출 순서 및 병렬 처리 전략, 응답 지연 시 타임아웃 기준
- **MVP 포함 여부:** Yes

## SCR-006: 콘텐츠 상세

- **진입 경로:** SCR-005 검색 결과 카드 탭
- **주요 UI 요소:**
  - 백 버튼
  - 포스터 이미지 (대형)
  - 작품명 / 원제 / 연도 / 장르
  - 줄거리 요약 (접기/펼치기)
  - 콘텐츠 유형별 정보 — 시리즈: 시즌 수, 총 에피소드 수 / 영화: 상영 시간
  - 출처 API 레이블
  - "라이브러리에 추가" CTA 버튼 (상단 고정 또는 하단 고정)
  - 이미 추가된 경우: 현재 감상 상태 배지 + "라이브러리에서 보기" 버튼
- **사용자 액션:**
  - "라이브러리에 추가" 탭 → SCR-007 (바텀 시트) 열기
  - "라이브러리에서 보기" 탭 → SCR-008 (현재 시청 중 상세)
  - 뒤로 가기 → SCR-005
- **다음 화면:**
  - "라이브러리에 추가" → SCR-007 (바텀 시트)
  - "라이브러리에서 보기" → SCR-008
- **엣지 케이스:**
  - 포스터 이미지 없음: 기본 플레이스홀더 이미지
  - 줄거리 없음: "줄거리 정보가 없습니다" 안내
  - 에피소드 수 정보 없음: "에피소드 정보를 불러오는 중..." 표시 후 로드 실패 시 "정보 없음"
  - 영화에 에피소드 선택 UI 노출 금지: 유형 판별 후 조건부 렌더링
  - 동일 작품이 두 API 출처로 라이브러리에 이미 있는 경우: [확실하지 않음 — 추후 검토] 중복 경고 처리 정책
- **MVP 포함 여부:** Yes

## SCR-007: My Library에 추가 (바텀 시트)

- **진입 경로:** SCR-006에서 "라이브러리에 추가" 버튼 탭
- **주요 UI 요소:**
  - 바텀 시트 핸들 (drag indicator)
  - 작품 썸네일 + 제목 (확인용)
  - 감상 상태 선택 목록:
    - "보고 싶음" (북마크 아이콘)
    - "보는 중" (재생 아이콘)
    - "완료" (체크 아이콘)
    - "드랍" (X 아이콘)
  - 취소 버튼
- **사용자 액션:**
  - 감상 상태 선택 → 즉시 저장 + 시트 닫힘
  - 바텀 시트 드래그 다운 또는 취소 버튼 → 닫힘
- **다음 화면:**
  - 저장 성공 → SCR-006 (버튼 상태 업데이트됨)
- **엣지 케이스:**
  - 저장 중 네트워크 오류: 오류 토스트 표시, 시트 유지
  - 저장 중 백 버튼 탭: 저장 완료까지 대기 (스피너 표시)
  - 이미 라이브러리에 있는 작품: 바텀 시트 열기 전에 "이미 라이브러리에 있습니다" 토스트 표시 후 SCR-008로 이동
- **MVP 포함 여부:** Yes

## SCR-008: 현재 시청 중 상세

- **진입 경로:** SCR-003 라이브러리 작품 카드 탭, 또는 SCR-006에서 "라이브러리에서 보기" 탭
- **주요 UI 요소:**
  - 백 버튼
  - 포스터 + 작품명 + 현재 감상 상태 배지
  - 감상 상태 변경 버튼 (현재 상태 탭 → 상태 선택 시트)
  - 시리즈: "에피소드 목록" 버튼
  - 전체 핀 목록 버튼 (해당 작품의 모든 핀)
  - 핀 요약 (최근 3개 핀 미리보기)
  - 진행률 표시 (시리즈: "3시즌 / 24화 중 12화 완료", 영화: "완료 / 미완료")

- **사용자 액션:**
  - 감상 상태 변경
  - "에피소드 목록" 탭 → SCR-009
  - "핀 전체 보기" 탭 → SCR-011 (작품 단위 필터)
  - 라이브러리에서 제거
- **다음 화면:**
  - SCR-009 (에피소드 선택)
  - SCR-011 (타임라인 핀 목록)
  - SCR-013 (태그별 핀 목록)
- **엣지 케이스:**
  - 영화 콘텐츠: 에피소드 목록 버튼 미표시
  - 핀이 없는 경우: "아직 핀이 없어요. 에피소드에서 첫 핀을 남겨보세요!" 빈 상태 UI
  - 시즌/에피소드 정보 로드 실패: 재시도 버튼 표시
- **MVP 포함 여부:** Yes

## SCR-009: 에피소드 선택

- **진입 경로:** SCR-008에서 "에피소드 목록" 버튼 탭
- **주요 UI 요소:**
  - 백 버튼
  - 시즌 선택 탭 (시즌 1 / 시즌 2 / ...)
  - 에피소드 목록 (번호, 제목, 방영일, 런타임)
  - 각 에피소드 행: 시청 완료 체크박스 / 체크 아이콘
  - 각 에피소드 행: 해당 에피소드의 핀 수 배지
  - 각 에피소드 행: "핀 추가" 버튼 (또는 탭 후 진입)
  - 시즌 진행률 표시 (예: "12 / 24화 완료")
- **사용자 액션:**
  - 시즌 탭 전환
  - 에피소드 체크박스 탭 → 완료/미완료 토글
  - "핀 추가" 버튼 탭 → SCR-010
  - 에피소드 행 탭 → SCR-011 (에피소드 단위 핀 목록)
- **다음 화면:**
  - SCR-010 (타임라인 핀 생성)
  - SCR-011 (타임라인 핀 목록, 에피소드 단위)
- **엣지 케이스:**
  - 에피소드 수가 매우 많은 경우 (200+화): FlashList 가상화 처리
  - 에피소드 제목 없음: "에피소드 [번호]" 기본 표시
  - 시즌이 1개뿐인 경우: 시즌 탭 미표시, 에피소드 목록 직접 표시
  - 에피소드 런타임 정보 없음: 런타임 필드 미표시
  - 마지막 에피소드 완료 체크: "전체 시청 완료! 감상 상태를 '완료'로 변경할까요?" 팝업
  - 영화 콘텐츠: 이 화면에 진입할 수 없음 (SCR-008에서 에피소드 버튼 미표시)
- **MVP 포함 여부:** Yes

## SCR-010: 타임라인 핀 생성 / 편집

- **진입 경로:**
  - 생성: SCR-009 에피소드 행의 "핀 추가" 탭
  - 편집: SCR-011 핀 카드의 편집 버튼 탭
- **주요 UI 요소:**
  - 백 버튼 (변경 감지 후 확인 팝업)
  - 화면 상단: 작품명 + 에피소드 번호 (컨텍스트)
  - 타임스탬프 입력 필드 (MM:SS 또는 HH:MM:SS 형식 힌트)
  - 타임스탬프 입력 오류 메시지 영역
  - 메모 입력 필드 (멀티라인, 최대 500자)
  - 태그 입력 필드 + 태그 칩 목록
  - 감정 선택 (수평 스크롤 감정 아이콘 목록)
  - 스포일러 토글
  - 저장 버튼 (헤더 우측 또는 하단 고정)
  - 삭제 버튼 (편집 모드 전용)
- **사용자 액션:**
  - 타임스탬프 입력
  - 메모 입력
  - 태그 입력 및 추가/제거
  - 감정 선택/해제
  - 스포일러 토글
  - 저장 버튼 탭
  - 삭제 버튼 탭 (편집 모드)
  - 뒤로 가기 (변경 미저장 경고)
- **다음 화면:**
  - 저장 완료 → SCR-011 (핀 목록) 또는 SCR-009 (에피소드 목록)로 복귀
  - 삭제 완료 → SCR-011
- **엣지 케이스:**
  - 타임스탬프 에피소드 런타임 초과: "에피소드 길이(X분)를 초과하는 시간값입니다" 오류 표시, 저장 불가
  - 에피소드 런타임 정보 없음: 타임스탬프 초과 검증 생략, 저장 허용
  - 동일 에피소드에 동일 시간값 핀 추가: 경고 없이 중복 허용
  - 메모 없이 저장 시도: [확실하지 않음 — 메모 필수 여부 추후 검토]
  - 저장 중 네트워크 오류: 오류 토스트, 입력 내용 유지
  - 변경 사항 있을 때 뒤로 가기: "저장되지 않은 내용이 있습니다. 나가시겠습니까?" 확인 팝업
  - 타임스탬프 자동 포맷: 숫자만 입력 시 자동으로 MM:SS 형식으로 변환 (09_timeline_pin_ux.md 참조)
- **MVP 포함 여부:** Yes — 핵심 기능

## SCR-011: 타임라인 핀 목록

- **진입 경로:**
  - SCR-008 "핀 전체 보기" 탭 (작품 단위)
  - SCR-009 에피소드 행 탭 (에피소드 단위)

  - 백 버튼

  - 태그 필터 칩 (수평 스크롤)
  - 핀 목록 (FlashList) — timestamp_seconds 오름차순
  - 각 핀 카드: 타임스탬프(HH:MM:SS 또는 MM:SS), 에피소드 번호, 메모 미리보기, 태그 칩, 감정 아이콘, 스포일러 배지

  - "핀 추가" FAB (Floating Action Button)
- **사용자 액션:**
  - 에피소드 필터 전환
  - 태그 필터 탭
  - 핀 카드 탭 → SCR-010 (편집)
  - 핀 카드 스와이프 또는 길게 누르기 → 삭제 옵션
  - 스포일러 핀 "보기" 탭 → 블러 해제
  - FAB 탭 → SCR-010 (핀 생성)
- **다음 화면:**
  - SCR-010 (타임라인 핀 생성/편집)
- **엣지 케이스:**
  - 핀 없음: "아직 핀이 없어요" 빈 상태 UI + FAB 강조
  - 동일 시간값 핀 여러 개: 모두 순서대로 나열
  - 에피소드 100개 이상 작품의 전체 핀 목록: 페이지네이션 또는 무한 스크롤 [Tech Lead 검토 필요]
  - 태그 필터 결과 0건: "해당 태그의 핀이 없습니다" 안내
  - 스포일러 핀 블러 해제 후 화면 재진입: 블러 상태로 초기화 [확실하지 않음 — 세션 내 유지 여부 추후 검토]
- **MVP 포함 여부:** Yes

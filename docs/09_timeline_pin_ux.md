# SceneNote — 타임라인 핀 UX 정책

**버전:** 1.0.0
**작성일:** 2026-05-02
**작성자:** MVP Product Planner
**상태:** 검토 중

---

## 개요

이 문서는 SceneNote의 핵심 기능인 **타임라인 핀(Timeline Pin)**의 UX 정책을 상세히 정의한다. 타임라인 핀은 사용자가 특정 에피소드의 특정 시간대에 감상 메모, 태그, 감정, 스포일러 여부를 기록하는 기능이다.

모든 타임스탬프는 `timestamp_seconds`(정수) 형태로 DB에 저장되며, 화면에서의 입력과 표시는 MM:SS 또는 HH:MM:SS 형식을 사용한다.

---

## 1. 핀 생성 플로우

### 1.1 진입 경로

```
에피소드 선택 화면 (SCR-009)
    → 에피소드 행의 "핀 추가" 버튼 탭
        → 핀 생성 화면 (SCR-010)
            → 타임스탬프 입력
            → 메모 입력 (선택)
            → 태그 입력 (선택)
            → 감정 선택 (선택)
            → 스포일러 토글 (선택)
            → 저장 버튼 탭
                → 유효성 검사
                    → 통과: 저장 완료 → 핀 목록으로 이동
                    → 실패: 오류 메시지 표시, 입력 유지
```

### 1.2 단계별 사용자 액션

| 단계 | 액션 | 필수 여부 |
|------|------|-----------|
| 1 | 에피소드 선택 화면에서 "핀 추가" 탭 | 필수 (진입) |
| 2 | 타임스탬프 입력 (MM:SS 또는 HH:MM:SS) | 선택 [확실하지 않음 — 타임스탬프 없는 핀 허용 여부 추후 검토] |
| 3 | 메모 입력 | 선택 |
| 4 | 태그 입력 | 선택 |
| 5 | 감정 선택 | 선택 |
| 6 | 스포일러 토글 | 선택 (기본: OFF) |
| 7 | 저장 버튼 탭 | 필수 |

### 1.3 영화 콘텐츠에서의 핀 생성

- 영화는 에피소드 개념이 없으므로 콘텐츠 상세 화면(SCR-008) 또는 별도 진입점에서 핀을 생성한다.
- [Tech Lead 검토 필요] 영화 핀을 위한 DB 레코드 구조 결정 필요 — `episode_id` 참조 없이 `content_id` 직접 참조 방식 또는 가상 에피소드 레코드 생성 방식 중 선택.

## 2. 시간값 입력 방식

### 2.1 지원 입력 형식

| 형식 | 예시 | 설명 |
|------|------|------|
| MM:SS | 14:32 | 14분 32초 (60분 미만 에피소드 기본) |
| HH:MM:SS | 1:23:45 | 1시간 23분 45초 (60분 이상 또는 영화) |

### 2.2 숫자만 입력 시 자동 포맷 규칙

사용자가 콜론(:) 없이 숫자만 연속 입력하는 경우 자동 포맷을 적용한다.

| 입력 | 자동 변환 결과 | 설명 |
|------|----------------|------|
| `1432` | `14:32` | 4자리 → MM:SS |
| `132345` | `1:23:45` | 6자리 → HH:MM:SS |
| `532` | `05:32` | 3자리 → 0M:SS로 해석 |
| `30` | `00:30` | 2자리 → 초(SS)로 해석 |
| `7` | `00:07` | 1자리 → 초(SS)로 해석 |

**자동 포맷 적용 시점:** 입력 필드 포커스 해제(blur) 시 또는 저장 버튼 탭 시.

**[Tech Lead 검토 필요]** 입력 중 실시간 포맷 변환 시 커서 위치 이동으로 인한 UX 혼란 가능성 검토. 저장 시 변환 방식이 더 안전할 수 있음.

### 2.3 입력 필드 UI 사양

- 키보드 타입: 숫자 패드 (numeric)
- 플레이스홀더: `00:00` (MM:SS 기본) 또는 `0:00:00` (HH:MM:SS)
- 최대 입력 길이: 8자 (HH:MM:SS + 콜론 2개)
- 콜론은 자동으로 삽입되거나 직접 입력 가능

### 2.4 타임스탬프 없이 메모만 입력하는 경우 처리

**MVP 정책:** [확실하지 않음 — 추후 검토]

두 가지 옵션이 있다:

| 옵션 | 설명 | 장점 | 단점 |
|------|------|------|------|
| A. 타임스탬프 필수 | 저장 불가, 오류 표시 | 데이터 일관성 유지 | 사용자 진입 장벽 증가 |
| B. 타임스탬프 선택 | `timestamp_seconds = null` 허용 | 유연한 메모 기록 가능 | 목록 정렬 로직 복잡도 증가 |

**현재 권장안 (추후 결정 필요):** 타임스탬프 없는 핀은 목록 맨 앞 또는 맨 뒤에 배치하는 방식으로 null을 허용하는 방향 검토.

## 3. 에피소드 길이 초과 시간값 처리

### 3.1 검증 시점

**MVP 정책: 저장 시 검증** (입력 중 실시간 검증 미수행)

- 입력 중에는 형식 오류만 실시간으로 피드백한다.
- 에피소드 길이 초과 여부는 저장 버튼 탭 시에 검증한다.

**이유:** 입력 중 실시간 초과 검증은 에피소드 런타임 데이터를 항상 보유하고 있어야 하며, 사용자 입력 중 오류 표시가 UX를 방해할 수 있다.

### 3.2 검증 조건

```
에피소드 런타임 정보 존재 AND 입력값 > 에피소드 런타임
    → 저장 불가, 오류 메시지 표시
```

### 3.3 오류 메시지

> "입력한 시간(HH:MM:SS)이 에피소드 길이(HH:MM:SS)를 초과했습니다."

- 타임스탬프 입력 필드 하단에 빨간 인라인 오류 메시지로 표시한다.
- 저장 버튼은 비활성화 상태로 변경된다.

### 3.4 에피소드 런타임 정보가 없는 경우

**정책: 검증 생략, 저장 허용**

- 에피소드 런타임 정보가 null 또는 0인 경우 초과 검증을 수행하지 않는다.
- 사용자 입력을 그대로 저장한다.
- 이 경우 타임스탬프 입력 필드 옆에 작은 주의 아이콘을 표시한다. [확실하지 않음 — UX 노출 여부 추후 검토]

### 3.5 초과 시간값 처리 흐름

```
저장 버튼 탭
    ↓
타임스탬프 형식 검증 (MM:SS / HH:MM:SS 유효성)
    ↓ 형식 오류 → 인라인 오류, 저장 중단
    ↓ 형식 정상
에피소드 런타임 정보 존재 여부 확인
    ↓ 런타임 없음 → 초과 검증 생략, 저장 진행
    ↓ 런타임 있음
timestamp_seconds > episode_duration_seconds 비교
    ↓ 초과 → 오류 메시지, 저장 중단
    ↓ 정상 → 저장 진행
```

## 4. 동일 에피소드 내 여러 핀 처리

### 4.1 중복 허용 정책

**핵심 원칙: 동일 에피소드 내 복수 핀을 제한 없이 허용한다.**

한 에피소드에서 여러 장면을 기록하는 것이 SceneNote의 핵심 사용 패턴이므로, 개수 제한 없이 허용한다.

### 4.2 UI 표시 방식

- 핀 목록에서 동일 에피소드의 핀은 `timestamp_seconds` 오름차순으로 나열된다.
- 에피소드 선택 화면(SCR-009)의 에피소드 행에 핀 개수 배지가 표시된다 (예: "3개 핀").
- 핀이 많은 에피소드 행은 시각적으로 강조한다 (배지 색상 변경 등).

### 4.3 정렬 순서

```
기본 정렬: timestamp_seconds ASC (오름차순)
동점 처리: created_at ASC (나중에 생성된 핀은 뒤에)
```

## 5. 같은 시간대 여러 핀 처리

### 5.1 중복 허용 정책

**핵심 원칙: 같은 timestamp_seconds 값의 핀을 제한 없이 허용한다.**

같은 장면에서 다른 감정이나 다른 메모를 남기고 싶을 수 있으므로, 충돌 없이 중복을 허용한다. 경고 메시지도 표시하지 않는다.

**근거:** 사용자가 같은 순간을 다른 시각으로 기록하는 행위는 유효한 사용 패턴이다.

### 5.2 UI 표시 방식

```
[핀 목록 예시 — 동일 시간값 14:32에 핀 2개]

 ● 14:32  에피소드 5
   "리바이가 등장하는 첫 장면"
   [명장면] [액션]  😮

 ● 14:32  에피소드 5
   "BGM이 너무 잘 어울림"
   [음악]  ❤️
```

- 동일 시간값 핀은 구분선 없이 연속으로 나열된다.
- 사용자가 두 핀이 같은 시간임을 인식할 수 있도록 타임스탬프를 동일하게 표시한다.
- 그룹핑(묶기) UI는 MVP에서 구현하지 않는다. Phase 2에서 검토.

## 6. 스포일러 핀 표시 방식

### 6.1 목록에서의 블러 처리

- `is_spoiler = true`인 핀의 **메모 텍스트 영역**을 블러(blur) 처리한다.
- 태그, 감정 아이콘, 타임스탬프는 블러 처리하지 않는다.
- 블러 처리된 영역 위에 "스포일러 포함" 레이블과 "보기" 버튼을 표시한다.

```
[스포일러 핀 카드 UI]
┌─────────────────────────────────┐
│ ● 23:15  에피소드 12             │
│ ████████████████████████  [보기] │  ← 메모 영역 블러
│  👁 스포일러  [명장면]  😱        │
└─────────────────────────────────┘
```

### 6.2 스포일러 해제 인터랙션

- "보기" 버튼 탭 시 해당 핀 카드의 메모 블러가 해제된다.
- 블러 해제는 **해당 세션(화면) 내에서만 유지**된다.
- 화면을 나갔다가 돌아오면 다시 블러 처리된 상태로 표시된다. [확실하지 않음 — 영구 해제 옵션 추후 검토]
- 핀 목록 전체의 스포일러를 한 번에 해제하는 "스포일러 모두 보기" 옵션은 Phase 2에서 검토.

### 6.3 핀 편집 화면에서의 스포일러

- 핀 편집 화면(SCR-010)에서는 스포일러 여부와 관계없이 메모 전체 내용이 표시된다.
- 편집 화면 진입 시 스포일러 핀임을 색상 또는 배너로 표시한다.

## 7. 핀 목록 UI 옵션

### 7.1 세로 리스트형 vs. 진행바형 비교

| 구분 | 세로 리스트형 | 진행바형 |
|------|--------------|---------|
| 설명 | 핀을 카드로 세로 나열 | 에피소드 진행바 위에 핀 점(dot) 표시 |
| 정보 표시량 | 많음 (메모, 태그, 감정 모두 보임) | 적음 (타임스탬프 위치만) |
| 직관성 | 중간 (시간 흐름 파악 어려움) | 높음 (시간 위치 직관적) |
| 구현 복잡도 | 낮음 | 높음 (에피소드 런타임 필수) |
| 스포일러 처리 | 쉬움 | 어려움 (점만 보이므로 블러 의미 없음) |
| 다수 핀 처리 | 스크롤로 해결 | 점 겹침 발생 |

### 7.2 MVP 선택 방식 및 이유

**MVP 선택: 세로 리스트형**

**이유:**
1. **구현 복잡도가 낮다.** 진행바형은 에피소드 런타임 데이터가 반드시 필요하며, 외부 API에서 런타임 정보가 없는 경우 UX가 깨진다.
2. **정보 밀도가 높다.** 메모, 태그, 감정을 한 화면에서 확인할 수 있다.
3. **스포일러 블러 처리가 자연스럽다.** 카드형에서 블러 영역 지정이 명확하다.
4. **동일 시간값 복수 핀 처리가 용이하다.** 진행바형에서는 같은 위치에 점이 겹쳐 표시된다.

**Phase 2에서:** 진행바형을 보조 뷰로 추가하여 사용자가 전환할 수 있도록 검토.

## 8. 핀 감정(Emotion) 옵션 정의

### 8.1 감정 목록

| 순서 | 감정 레이블 | 이모지 | 설명 |
|------|------------|--------|------|
| 1 | 감동 | 😭 | 눈물이 날 만큼 감동적인 장면 |
| 2 | 설렘 | 🥰 | 두근거리는 장면 (로맨스 등) |
| 3 | 긴장 | 😰 | 손에 땀을 쥐게 하는 긴장감 |
| 4 | 놀람 | 😮 | 예상치 못한 반전 또는 충격 |
| 5 | 즐거움 | 😄 | 유쾌하고 웃긴 장면 |
| 6 | 슬픔 | 😢 | 슬픔 또는 안타까움 |
| 7 | 분노 | 😤 | 캐릭터의 행동이나 상황에 대한 분노 |
| 8 | 공포 | 😱 | 무섭거나 충격적인 장면 |
| 9 | 공감 | 🥺 | 깊은 공감이 가는 장면 |
| 10 | 최고 | 🔥 | 최고의 명장면, 압도적인 퀄리티 |

### 8.2 감정 선택 UI 규칙

- 감정은 최대 **1개만** 선택 가능하다.
- 이미 선택된 감정을 다시 탭하면 **선택 해제**된다.
- 선택된 감정은 강조 테두리 또는 배경색으로 구분된다.
- 감정 목록은 수평 스크롤 형태로 표시된다.
- 핀 목록 카드에서 감정 이모지가 작게 표시된다.

### 8.3 감정 목록 확장 정책

- Phase 2에서 사용자 피드백을 반영하여 감정 목록을 추가/제거할 수 있다.
- 감정 목록 변경 시 기존 핀의 감정 데이터는 영향받지 않는다.

## 9. 태그 입력 방식

### 9.1 자유 입력 방식 (MVP)

**MVP 정책: 자유 텍스트 입력 허용**

- 사용자가 원하는 텍스트를 태그로 자유롭게 입력한다.
- 태그 입력 후 Enter, 쉼표(,), 또는 스페이스(띄어쓰기) 입력 시 태그가 칩(chip)으로 확정된다.
  - **[확실하지 않음 — 스페이스를 구분자로 사용할지 추후 검토]** 스페이스를 구분자로 사용하면 여러 단어 태그 입력이 불가하다.
- 확정된 태그는 칩 형태로 입력 필드 아래 또는 같은 줄에 표시된다.
- 태그 칩의 X 버튼 탭 시 해당 태그가 제거된다.

### 9.2 추천 태그 (P2)

- MVP 이후 사용자가 이전에 사용한 태그 목록을 자동완성으로 제안한다.
- 입력 중 기존 태그와 일치하는 후보가 드롭다운으로 표시된다.
- 추천 태그 선택 시 즉시 칩으로 추가된다.

### 9.3 태그 규칙

| 항목 | 정책 |
|------|------|
| 최대 길이 | 태그 하나당 20자 이하 [확실하지 않음 — 추후 결정] |
| 최대 개수 | 핀 하나에 최대 10개 [확실하지 않음 — 추후 결정] |
| 대소문자 | 그대로 저장 (대소문자 구분) |
| 중복 태그 | 동일 핀에 같은 태그 중복 추가 방지 |
| 허용 문자 | 한국어, 영문, 숫자, 특수문자 모두 허용 [확실하지 않음 — 추후 검토] |

## 10. 핀 수정/삭제 정책

### 10.1 수정 정책

| 항목 | 정책 |
|------|------|
| 수정 가능 시점 | 핀 생성 후 언제든지 |
| 수정 가능 항목 | 타임스탬프, 메모, 태그, 감정, 스포일러 여부 모두 수정 가능 |
| 수정 불가 항목 | 연결된 에피소드 (에피소드 변경 불가) |
| 수정 후 정렬 | 타임스탬프 변경 시 목록이 재정렬됨 |
| 미저장 나가기 | "저장되지 않은 내용이 있습니다. 나가시겠습니까?" 확인 팝업 표시 |

**에피소드 변경 불가 이유:** 에피소드 간 핀 이동은 핀의 컨텍스트를 훼손할 수 있다. 필요한 경우 기존 핀을 삭제하고 올바른 에피소드에 새로 생성하도록 유도한다.

### 10.2 삭제 정책

| 항목 | 정책 |
|------|------|
| 삭제 방법 | 핀 카드 편집 화면의 삭제 버튼, 또는 목록에서 스와이프 삭제 |
| 삭제 전 확인 | "이 핀을 삭제하시겠습니까?" 확인 팝업 필수 |
| 삭제 범위 | `timeline_pins` 레코드 + `timeline_pin_tags` 레코드 동시 삭제 |
| 삭제 취소 | 확인 팝업에서 취소 선택 시 삭제 중단 |
| 실행 취소 (Undo) | MVP에서 제공하지 않음. Phase 2에서 검토. |
